# BugForge

BugForge is an OpenEnv environment for training and evaluating agents on a real software engineering task: debugging Python code. Each episode injects exactly one hidden bug into a task-specific codebase. The agent must run tests, inspect source files, apply an exact patch, and terminate when the issue is fixed.

## Why This Environment Matters

BugForge scores the full debugging trajectory, not just final correctness:

- gather evidence with tests
- inspect relevant files quickly
- apply precise patches
- stop efficiently once solved

This makes it useful for RL training and evaluation of code-debugging agents.

## Action Space

`BugforgeAction` supports:

- `{"type": "run_tests"}`
- `{"type": "read_file", "file": "filename.py"}`
- `{"type": "apply_patch", "file": "filename.py", "old_code": "...", "new_code": "..."}`
- `{"type": "done"}`

## Observation Space

`BugforgeObservation` contains:

- `output`
- `tests_passing`
- `tests_total`
- `files_read`
- `steps_remaining`
- `patches_applied`

Step results also include `reward` and `done`.

## Tasks

BugForge currently ships with 5 deterministic tasks:

| Task | Difficulty | Bug Location | Objective |
|---|---|---|---|
| `task_1` | Easy | `utils.py` | Fix arithmetic divisor bug |
| `task_2` | Medium | `models.py` | Fix cross-file type mismatch |
| `task_3` | Hard | `cart.py` | Restore missing coupon fallback branch |
| `task_4` | Medium | `utils.py` | Fix username normalization trim bug |
| `task_5` | Hard | `models.py` | Fix shipping-zone misclassification |

All tasks reset from clean templates and inject exactly one bug from config.

## Reward and Grading

Per-step rewards (environment-side):

- `run_tests`: `0.10` first call, then `0.05`
- `read_file`: `0.15` when reading the true bug file first time, else `0.05`
- `apply_patch`: `1.00` if all tests pass, `0.30` for partial improvement, else `0.00`
- `done`: `1.00` when all tests pass, else `0.00`

Final score (`grade()`):

- solved: `0.8` to `1.0` based on efficiency
- partially solved: up to `0.6` based on passed tests
- bug-file discovery fallback: `0.2`

## Recent Improvements

- test parsing is now robust to more exception types and does not rely on counting only a fixed subset of errors
- test totals are discovered dynamically from each task's `tests.py` rather than hardcoded assumptions
- grading remains in the same `0.0` to `1.0` range and keeps the original reward design semantics

## Baseline Inference

Baseline script: `inference.py` (root).

It:

- uses OpenAI Python client
- reads `HF_TOKEN`, `API_BASE_URL`, `MODEL_NAME`, `HF_SPACE_URL`
- emits required `[START]`, `[STEP]`, `[END]` logs
- is designed to run under 20 minutes on a valid model/provider setup

Baseline performance is model/provider dependent and should be regenerated by running `python inference.py`.

Mandatory runtime variables:

- `HF_TOKEN`
- `API_BASE_URL`
- `MODEL_NAME`
- `HF_SPACE_URL`

Required stdout format (exact line types):

- `[START] task=<task_name> env=<benchmark> model=<model_name>`
- `[STEP] step=<n> action=<action_str> reward=<0.00> done=<true|false> error=<msg|null>`
- `[END] success=<true|false> steps=<n> score=<score> rewards=<r1,r2,...,rn>`

## Setup

```bash
pip install -e .
```

```powershell
$env:HF_TOKEN = "<your_api_key>"
$env:API_BASE_URL = "https://router.huggingface.co/v1"
$env:MODEL_NAME = "Qwen/Qwen2.5-72B-Instruct"
$env:HF_SPACE_URL = "https://pranayteja31-BugForge.hf.space"
```

## Run Locally

```bash
python -m bugforge.server.app
python inference.py
```

## Docker

```bash
docker build -t bugforge-env .
docker run --rm -p 8000:8000 bugforge-env
```

## Validation

```bash
./validate.sh https://pranayteja31-BugForge.hf.space .
```

Expected checks:

- Space responds to `POST /reset`
- Docker build succeeds
- `openenv validate` passes
